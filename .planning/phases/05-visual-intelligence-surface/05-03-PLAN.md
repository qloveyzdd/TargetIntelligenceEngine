<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chart/graph click -> shared explanation panel | 不同入口的点击结果必须映射到同一解释 schema，否则会出现双重真相 |
| persisted run -> reopened visual surface | reopen 后图表必须继续读取 persisted run，而不是客户端临时缓存 |
| client-only chart libraries -> workspace shell | 依赖 DOM 的图组件必须隔离在 client-only 边界，避免 SSR/hydration 破坏单页工作台 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | `buildVisualExplanation()` | mitigate | 所有解释内容只来自 `run.scoring`、`run.evidence`、`run.candidates` 与 graph target，不拼装新的业务结论。 |
| T-05-08 | Information Disclosure | `VisualExplanationPanel` | mitigate | 面板只展开当前 target 相关 evidence 摘要与 excerpt，主图不渲染 Evidence 节点，不泄漏整批原始对象。 |
| T-05-09 | Elevation of Privilege | client-only chart components | mitigate | 用 `next/dynamic(..., { ssr: false })` 隔离 ECharts/React Flow，禁止它们在服务端路径执行浏览器专属逻辑。 |
</threat_model>
