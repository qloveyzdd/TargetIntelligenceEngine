<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| extracted evidence -> stable `evidence.id` | 证据一旦没有稳定标识，后续全部解释链都会变脆 |
| scoring service -> OpenAI assessment output | 单条 evidence assessment 进入本地聚合前需要校验和约束 |
| repository updates -> cached `run.scoring` | 上游数据变化后若不清空缓存，会泄露陈旧评分 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03 | Integrity | stable evidence identity | mitigate | 通过 `assignEvidenceId()` 固化 `evidence.id`，并用 schema/extract 单测保证 legacy 与新数据都能稳定回填 |
| T-04-04 | Integrity | unknown / overall 聚合 | mitigate | 在 `buildScoringSnapshot()` 中显式建模 `status: "unknown"`、`score: null`、`coverage` 与 `unknownCount`，用单测锁死行为 |
| T-04-05 | Repudiation | stale scoring cache | mitigate | 把评分失效逻辑集中在 repository helper，任何影响评分的上游字段更新都清空 `run.scoring` |
</threat_model>
